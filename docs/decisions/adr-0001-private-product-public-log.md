# ADR-0001: 製品コードと公開ログを分ける

## 現在の状態

採用

## 考えたこと

Trulay は製品化を見据えているため、本体コードや詳細仕様をすべて公開することにはリスクがあります。

一方で、開発プロセスや技術選定、設計判断は、公開可能な範囲で記録することで、開発の進め方や判断の流れを伝えやすくなります。

## 決めたこと

- 製品コード本体は非公開リポジトリで管理する
- 公開可能な開発ログと技術メモは公開リポジトリで管理する

## そうした理由

- 製品コードや詳細仕様を必要以上に公開せずに済む
- 開発プロセスや技術選定の記録を残せる
- 設計判断や進め方の流れを整理できる
- 公開する情報と公開しない情報の線引きを明確にできる
