# ADR-0001: 製品コードと公開ログを分ける

## ステータス

採用

## 背景

Trulay は製品化を見据えているため、本体コードや詳細仕様をすべて公開することにはリスクがあります。

一方で、開発プロセス、技術選定、学習、設計判断はポートフォリオとして公開する価値があります。

## 決定

- 製品コード本体は private repository で管理する
- 公開可能な開発ログと技術メモは public repository で管理する

## 理由

- 製品の中核を守れる
- 技術力や継続力を外部に示せる
- 設計やPL的な進め方を見せられる
- 公開範囲を明確にできる
