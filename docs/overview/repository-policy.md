# Repository Policy

Trulay は製品化を見据えて、本体コードと公開ログを分けて管理します。

## Private Repository

private repository では、以下を管理します。

- 製品コード本体
- 詳細な内部仕様
- DB設計の詳細
- 実装に直結する設計書
- セキュリティ上公開すべきでない情報

## Public Repository

この public repository では、以下を管理します。

- 開発ログ
- 技術選定の理由
- 設計判断の概要
- 学習メモ
- 公開可能なサンプルコード

## 公開方針

公開する情報は、製品の中核や機密情報を含まない粒度に整理します。

公開リポジトリでは、完成物のすべてを見せることよりも、設計の考え方、進め方、技術判断の過程が伝わることを重視します。
