# リポジトリ公開方針

Trulay は製品化を見据えて、本体コードと公開ログを分けて管理します。

## 非公開リポジトリ

非公開リポジトリでは、以下を管理します。

- 製品コード本体
- 詳細な内部仕様
- DB設計の詳細
- 実装に直結する設計書
- セキュリティ上公開すべきでない情報

## 公開リポジトリ

この公開リポジトリでは、以下を管理します。

- 開発ログ
- 技術選定の理由
- 設計判断の概要
- 学習メモ
- 公開可能なサンプルコード

## 公開方針

公開する情報は、製品コードや機密情報を含まない範囲に整理します。

公開リポジトリでは、完成物のすべてを公開することよりも、設計の考え方、進め方、技術判断の過程が伝わることを重視します。
